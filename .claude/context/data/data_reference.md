### Issue 6 Mitigation ‚Äî Unlabelled Annotation Events

#### Context
- 2,739 annotation events across 22 recordings
- 550 events have a species/common name tag (20%)
- 2,189 events are **score-only** (machine-detected acoustic events, no label)
- Only 1 scientific species labelled: *Ninox boobook* / Southern Boobook (439 events)
- 17 common names total, all with very low counts

#### Proposed Mitigation

**Tier 1 ‚Äî Auto-labelling with BirdNET (recommended for MVP)**
Run BirdNET (Cornell Lab pre-trained bird species classifier) over all 6,148 clips.
BirdNET outputs species + confidence score per 3-second window.
- Dramatically increases species coverage from 17 ‚Üí potentially 100+ species
- Operates directly on audio, no manual effort
- Threshold at ‚â• 0.7 confidence for reliable pseudo-labels
- Output merges with existing A2O annotations (A2O labels take precedence)

```bash
# After webm ‚Üí wav conversion (Issue 1)
python3 -m birdnet_analyzer --input downloaded_clips/ --output birdnet_labels/ --min_conf 0.7
```

**Tier 2 ‚Äî Treat score-only events as "bioacoustic activity" binary label**
The 2,189 unlabelled events all have score ‚â• 0.50 (confirmed). They represent machine-detected acoustic activity (not silence / wind / rain). Use them as a binary `activity=1` label rather than discarding.
- Adds weak supervision signal at no extra cost
- Useful for training an activity-detection head alongside the generative model

**Tier 3 ‚Äî Unsupervised for generative training (current default)**
For the soundscape generation and transformation pipelines, species labels are NOT required. The model learns the mapping `audio embedding + env conditions ‚Üí soundscape`. Annotations are only needed for evaluation of ecological plausibility. Proceed without them for Stage 2 pilot.

#### Recommended approach for MVP
Use **Tier 3** for training, **Tier 1** (BirdNET) for post-hoc evaluation of generated soundscapes. Tier 2 can be added to the dataloader with minimal effort.

### Notes for AI Training

- `cloud_clearness_index` is empty for nighttime recordings (dawn + night bins) ‚Äî treat as a masked/conditional feature, not fill with 0.
- `days_since_rain` uses UTC date throughout to stay consistent with NASA POWER key space. Values reflect drought conditions up to 90 days prior.
- `sunrise_local` / `sunset_local` are AEST `HH:MM` strings ‚Äî convert to decimal hours for model input.
- Temporal features (`month`, `day_of_year`, `season`) should be encoded cyclically (sin/cos) to preserve continuity across year/day boundaries.
- `--ndvi` (MODIS NDVI vegetation index) is listed as a future feature ‚Äî not yet fetched.
