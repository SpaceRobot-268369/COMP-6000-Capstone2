## Issue 6 Mitigation — Unlabelled Annotation Events

### Context
- 2,739 annotation events across 22 recordings
- 550 events have a species/common name tag (20%)
- 2,189 events are **score-only** (machine-detected acoustic events, no label)
- Only 1 scientific species labelled: *Ninox boobook* / Southern Boobook (439 events)
- 17 common names total, all with very low counts

### Proposed Mitigation

**Tier 1 — Auto-labelling with BirdNET (recommended for MVP)**
Run BirdNET (Cornell Lab pre-trained bird species classifier) over all 6,148 clips.
BirdNET outputs species + confidence score per 3-second window.
- Dramatically increases species coverage from 17 → potentially 100+ species
- Operates directly on audio, no manual effort
- Threshold at ≥ 0.7 confidence for reliable pseudo-labels
- Output merges with existing A2O annotations (A2O labels take precedence)

```bash
# After webm → wav conversion (Issue 1)
python3 -m birdnet_analyzer --input downloaded_clips/ --output birdnet_labels/ --min_conf 0.7
```

**Tier 2 — Treat score-only events as "bioacoustic activity" binary label**
The 2,189 unlabelled events all have score ≥ 0.50 (confirmed). They represent machine-detected acoustic activity (not silence / wind / rain). Use them as a binary `activity=1` label rather than discarding.
- Adds weak supervision signal at no extra cost
- Useful for training an activity-detection head alongside the generative model

**Tier 3 — Unsupervised for generative training (current default)**
For the soundscape generation and transformation pipelines, species labels are NOT required. The model learns the mapping `audio embedding + env conditions → soundscape`. Annotations are only needed for evaluation of ecological plausibility. Proceed without them for Stage 2 pilot.

### Recommended approach for MVP
Use **Tier 3** for training, **Tier 1** (BirdNET) for post-hoc evaluation of generated soundscapes. Tier 2 can be added to the dataloader with minimal effort.

## Training Manifest — `site_257_training_manifest.csv`

**6,148 rows** — one per clip file. Generated by `script/build_training_manifest.py`.

### Columns

| Column | Description |
|--------|-------------|
| `clip_path` | Relative path to `.webm` file from project root |
| `recording_id` | Links to `site_257_filtered_items.csv` and `site_257_env_data.csv` |
| `clip_index` | 1-based clip number within the recording |
| `clip_start_seconds` | Start offset in the original recording (s) |
| `clip_end_seconds` | End offset (s) |
| `clip_duration_seconds` | Actual clip length (≤ 300 s) |
| `sample_bin` | Diel bin: `dawn` / `morning` / `afternoon` / `night` |
| `sample_local_date` | AEST local date (YYYY-MM-DD) |
| *(+20 env columns)* | All features from `site_257_env_data.csv` |
